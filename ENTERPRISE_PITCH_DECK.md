# DemandOrchestrator — Enterprise Pitch Deck (Draft)

## 1) The Problem
Enterprise social teams are blocked by tool sprawl, unclear approvals, and publishing risk.

## 2) Why Existing Tools Fail
Most schedulers optimize for posting volume, not governance:
- weak role controls
- no per-account publish authority
- poor auditability

## 3) Our Thesis
**Speed + Control** in one workflow:
idea → generate → review → approve → publish (with policy enforcement)

## 4) Built for Enterprise Identity
- Corporate SSO path
- Corporate-domain gating
- Workspace-scoped access model

## 5) Governance as a Product Feature
- RBAC (Owner/Admin/Publisher/Reviewer/Contributor)
- Per-platform authorized publisher matrix
- Approval workflows before publish

## 6) Risk & Compliance Advantage
- Fail-closed publish behavior
- Structured audit trail for publish decisions
- Platform guardrails (e.g., LinkedIn org-only posture)

## 7) ROI Story
- Faster campaign throughput
- Fewer publishing errors
- Lower brand/compliance risk
- Clear operator accountability

## 8) Plan Design
### Personal (lower cost)
- single user
- essential scheduling/publishing
- no enterprise governance overhead

### Corporate (premium)
- multi-seat org access
- SSO + domain controls
- RBAC + approval + audit + per-account authorization

## 9) Performance Brain (How It Adapts Automatically)
Closed-loop engine:
- tag every post at creation (topic, hook, format, CTA, publish window)
- ingest platform outcomes (retention, shares, saves, clicks, conversions)
- compute unified north-star score per post/account
- rebalance next-batch generation weights toward winning patterns

Automatic actions:
- scale winners (auto-generate variants)
- remix borderline performers (new hook/CTA/length)
- suppress weak patterns (reduced priority / no boost)

## 10) Premium Faceless Routing Strategy
Two-lane model policy:
- lane A (cost): trend ingest, ideation, first-draft scripts on low-cost models
- lane B (quality): premium cinematic/faceless render models for shortlisted scripts only

Routing gate before premium render:
- hook score >= threshold
- narrative coherence >= threshold
- visual beat-map completeness >= threshold
- policy/brand safety pass

Why this wins:
- lower average generation cost
- premium visual output where it matters
- consistent quality under volume

## 11) Voice DNA (Per-Workspace Uniqueness)
- onboarding generates a unique voice seed per workspace
- style packs (Authority/Explainer/Story) are parameterized, not shared clones
- each customer gets distinct pacing/energy/pause emphasis profile
- performance loop tunes voice profile over time

Business impact:
- avoids platform sameness
- preserves brand distinctiveness at scale

## 12) Pilot Proposal (2–4 weeks)
- connect 1–2 channels
- onboard core team
- enforce approval + authorization policies
- measure time-to-publish + error rate
- measure adaptive lift from loop tuning (share/save/click/conversion deltas)
- measure COGS delta from two-lane routing

## 12) CTA
Run a guided pilot and security review.
Success = faster content ops without sacrificing control.
