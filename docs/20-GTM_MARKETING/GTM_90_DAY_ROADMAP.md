# DemandOrchestrator — 90-Day GTM Roadmap (Draft v1)

Date: 2026-03-07
Owner: DO Core Team

## Goals (90 days)
1. Validate production reliability across all content modes.
2. Improve discoverability in traditional search + AI answer engines (Gemini, ChatGPT, Claude, Grok surfaces).
3. Build repeatable top-of-funnel + sales conversion motion.
4. Define App Store decision gate and MVP scope.

---

## Phase 0 (Immediate: Days 1–3) — Production Readiness Gate

### Track A — Content Mode Quality/Reliability
- Avatar mode: background-template consistency, lip-sync quality, render completion reliability.
- Faceless mode: scene/script alignment and render stability.
- Image mode: quality/brand consistency.
- Text mode: originality + channel fit.
- Cross-mode queue/review/schedule reliability.

### Exit Criteria
- No P0/P1 blockers in core demo paths.
- Stable end-to-end generation for each mode.
- Known issue register with owners + ETAs.

---

## Phase 1 (Days 4–30) — Foundation (SEO + AEO + Conversion)

### 1) Positioning + Messaging System
- Lock core category line: “AI-assisted content engine”.
- Create message hierarchy:
  - Promise
  - Proof
  - Persona/use-case variants
- Build approved phrase bank for consistency across site/docs/social.

### 2) Website/Docs Discoverability Foundation
- Build/refresh pages:
  - /features
  - /use-cases (agency, founder-led, local business)
  - /compare (alternatives)
  - /pricing
  - /security/governance
- Structured data (Organization, Product, FAQ, HowTo where relevant).
- Technical SEO hygiene: metadata, canonical, sitemap freshness, internal links.

### 3) AI Answer Engine Readiness (AEO/GEO)
- Publish “best answer” pages for high-intent prompts:
  - best AI content engine for multi-format publishing
  - avatar + faceless in one workflow
  - social workflow with review + governance
- Ensure concise, citation-friendly definitions and clear product claims.
- Expand docs with short answer blocks + proof snippets.

### 4) Conversion Layer
- Demo CTA architecture:
  - “Watch 90-sec demo”
  - “Start guided onboarding”
- Add trust blocks:
  - capabilities matrix
  - reliability posture
  - transparent AI-assisted language
- Ship a platform/channel matrix so users understand available content modes per destination.

### Phase 1 KPIs
- Indexed page growth
- Branded query growth
- Demo CTA CTR
- Onboarding completion rate

---

## Phase 2 (Days 31–60) — Demand Creation + Sales Motion

### 1) Content Engine for Demand Creation
- Weekly cadence:
  - 2 educational posts
  - 1 product proof post
  - 1 founder POV post
  - 1 short demo clip
- Build 3 case-style narratives (even pre-revenue if needed):
  - before/after workflow speed
  - quality consistency
  - multi-format throughput

### 2) Sales Enablement
- Create core sales assets:
  - one-pager
  - 90-sec hero demo
  - deep-dive demo
  - objection handling script
- Build outbound target list + ICP prioritization.
- Define discovery call script with qualifying criteria.

### 3) AI Discoverability Boosters
- Publish guest posts/partner citations.
- List on relevant directories/communities.
- Encourage third-party references using consistent category language.

### Phase 2 KPIs
- Qualified demo requests/week
- Lead-to-demo conversion
- Demo-to-trial/activation conversion
- Mention/citation growth across external sources

---

## Phase 3 (Days 61–90) — Scale + App Store Decision

### 1) Funnel Optimization
- Optimize onboarding friction points with analytics.
- Tighten helper triggers + guided assistance for stuck users.
- Improve trial-to-activation path.

### 2) App Store Track (Decision Gate)

#### Gate Criteria (must pass)
- Web activation + retention trend is stable.
- Clear mobile-specific use case validated (review/approve/alerts/queue edits).
- Team capacity available for App Store release + maintenance.

#### If pass: iOS Companion MVP
- Scope:
  - notifications
  - review/approve/reject
  - queue status
  - light edits
- Not full production studio initially.

### Phase 3 KPIs
- Activation and retention improvements
- Sales cycle compression
- App Store go/no-go decision documented

- Thursday: sales outreach + demo iteration
